Getting Started

Introduction

Syncra is an offline-first sync engine that lets you build applications that work seamlessly without a network connection. It manages local state in IndexedDB, queues operations when offline, and automatically syncs to your backend when connectivity is restored.

The SDK handles conflict resolution, exponential backoff retries, and delta synchronization so you can focus on building features instead of managing network state.

Installation

Install the Syncra SDK via npm or your preferred package manager:

terminal
$ npm install syncra-sdk

Quick Start

Initialize the SDK once at your app entry point:

syncra-init.ts 
import { SyncraSDK } from 'syncra-sdk';

const sdk = new SyncraSDK({
  baseUrl: 'https://api.syncra.dev',
  token: 'your-jwt-token',
  syncInterval: 30000,
});

await sdk.initialize();
await sdk.createRecord({ title: 'My note' });

sdk.on('sync-complete', ({ applied }) => {
  console.log(`Synced ${applied} records`);
});

SDK Reference

SyncraSDK

The main class. Instantiate once and reuse throughout your application.

new SyncraSDK(options: SyncraOptions)
Option Type Description
baseUrlstringYour Syncra API endpoint
tokenstringJWT auth token
syncIntervalnumberSync interval in ms (default: 30000)

createRecord

Creates a new record locally and queues it for sync. Returns immediately — no network required.

sdk.createRecord(data): Promise<SyncRecord>

sync()

Manually trigger a sync cycle. Pushes queued operations and pulls remote updates.

sdk.sync(): Promise<SyncResult>

Events

Subscribe using sdk.on(event, handler).

sync-complete

Fired after a successful sync. Payload: { applied, pushed }

conflict

Fired when a conflict is detected. Payload: ConflictEvent

online

Fired when connectivity is restored.

offline

Fired when the device goes offline.

API Reference

Authentication

All API requests require a Bearer token in the Authorization header.

Authorization: Bearer <your-jwt-token>

POST /sync

Push a batch of queued operations to the server. Processed atomically with idempotency keys.

request.json 
{
  "operations": [{
    "id": "op_abc123",
    "type": "create",
    "data": { "title": "My note" },
    "timestamp": 1704067200000
  }]
}

GET /sync/updates

Fetch remote updates since a given timestamp. Returns delta records only.

GET /sync/updates?since=1704067200000
